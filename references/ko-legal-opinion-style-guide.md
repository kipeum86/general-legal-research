# Korean Legal Opinion (Memorandum) Style Guide

> **목적:** 한국 법무법인 수준의 대외용 법률 검토 의견서(Memorandum)를 AI 에이전트가 일관되게 생성할 수 있도록, 실제 법무법인 작성 의견서 3건에서 추출한 스타일 규칙을 체계화한 가이드.
>
> **적용 범위:** 한국어 법률 의견서, 법률 검토 메모, 클라이언트 대면 법률 문서 전반.
>
> **이식성:** 이 문서는 self-contained — 외부 파일 의존 없이 이 문서만으로 스타일 재현이 가능합니다. 다른 에이전트 프로젝트에 `references/` 또는 유사 경로에 복사하여 사용할 수 있습니다.

---

## 1. Document Architecture (문서 전체 구조)

한국어 법률 의견서는 다음 순서로 구성됩니다:

```
┌─────────────────────────────────────┐
│  1. 문서 유형 표시 (MEMORANDUM)       │
│  2. 작성 일자                         │
│  3. 정보 블록 (수신/발신/제목)          │
│  4. 배경 사실 관계                     │
│  5. 질의의 요지                       │
│  6. 법률 의견의 한계 (Disclaimer)      │
│  7. 검토의견 (본론)                    │
│     ├─ 관련 법령                      │
│     ├─ 관련 판례 (해당 시)             │
│     └─ 적용 및 분석                   │
│  8. 결론                             │
│  9. 종결 Disclaimer                  │
│ 10. 서명 블록                         │
│ 11. 각주 (해당 시)                    │
└─────────────────────────────────────┘
```

### 핵심 원칙
- **질의 중심 구조:** 의뢰인의 질의 사항이 문서의 뼈대를 결정합니다. 질의가 3개면 검토의견도 3개 파트로 나뉩니다.
- **법령 → 판례 → 적용 순서:** 각 쟁점마다 관련 법령을 먼저 제시하고, 판례가 있으면 그 다음, 마지막으로 본건 사안에 적용합니다.
- **결론 선행 가능:** 단순 쟁점은 결론을 먼저 밝히고 근거를 전개할 수 있습니다. 복합 쟁점은 분석 후 결론을 도출합니다.

## 2. Header & Information Block (헤더/정보 블록)

### 2.1 문서 유형 표시

```
**MEMORANDUM**
```

- 볼드체, 중앙 정렬 또는 좌측 정렬.
- 영문 "MEMORANDUM" 사용이 관례 (한국 대형 법무법인 표준).

### 2.2 작성 일자

```
2025. 2. 21.
```

- 형식: `YYYY. M. D.` (마침표 구분, 월·일은 앞에 0 붙이지 않음)
- 문서 최상단, MEMORANDUM 바로 아래 위치.

### 2.3 정보 블록

테이블(박스) 형태로 구성:

```
┌──────────────────────────────────────────────┐
│ 수 신 : [의뢰인 법인명/기관명]                   │
│ 참 조 : [담당자명 직함] (선택 — 있을 때만)        │
│ 발 신 : 법무법인 [법인명]                        │
│ 제 목 : [의견서 주제]                            │
└──────────────────────────────────────────────┘
```

**규칙:**
- `수 신`, `참 조`, `발 신`, `제 목` — 볼드체, 글자 사이 한 칸 띄어쓰기 (시각적 정렬용).
- `참 조` 항목은 선택적. 특정 담당자가 있을 때만 포함.
- `제 목`은 문서의 핵심 주제를 간결하게. 예: "변상금 납부 의무 관련 법률 검토 의견서"

## 3. Body Structure Patterns (본문 구조 패턴)

### 3.1 배경 사실 관계 (Background Facts)

- 섹션 제목: `질의의 배경`, `배경 사실 관계`, 또는 `검토의 배경 및 질의의 요지`
- 볼드체 번호 매김으로 시작: **1. 질의의 배경**
- 사실관계는 아라비아 숫자(1, 2, 3...)로 순차 기술.
- 시간순(chronological) 서술이 기본.
- 타임라인이 중요한 경우 bullet list로 별도 정리:

```
* 2025. 8. 27. : 귀 협회의 「올해의 벤처상」 공고
* 2025. 9. 25. : 표창대상자의 신청서 제출
* 2025. 10. 3. ~ 10. 8. : 기본요건 심사
```

- 당사자·대상을 최초 언급 시 정의 용어로 지정 (→ § 6 참조).

### 3.2 질의의 요지 (Questions Presented)

- 의뢰인의 구체적 질의 사항을 명확히 기술.
- 질의가 여러 개인 경우 번호를 매겨 열거:

```
귀사는 다음과 같은 사항을 질의하셨습니다.
1. 본건 조합의 결성 및 본건 출자에 대한 관련 법규상 제한 여부
2. 본건 출자에 대한 이사회 결의, 공시 등 필요한 절차
3. 사전에 범위를 정하여 포괄적으로 승인받는 것이 가능한지 여부
```

### 3.3 법률 의견의 한계 (Limitations Disclaimer)

검토의견 시작 직전에 배치:

```
아래 의견은 귀 협회가 제공한 자료 및 정보만을 전제로 귀 협회가 문의한
사항에 국한된 법률검토임을 말씀드립니다. 이 사건에 제공된 자료 및 정보
이외에 다른 특별한 사정이 있는 경우 그 법률적 판단이 달라질 수 있습니다.
```

**핵심 요소:**
- 제공된 자료만을 전제로 한다는 점
- 문의 사항에 국한된다는 점
- 다른 사정이 있으면 판단이 달라질 수 있다는 점

### 3.4 검토의견 (Analysis)

검토의견은 쟁점별로 다음 하위 구조를 반복합니다:

```
검토의견
├─ [쟁점 1]
│   ├─ 관련 법령 (법령 블록)
│   ├─ 관련 판례 (해당 시)
│   └─ 검토 의견 (본건 적용)
├─ [쟁점 2]
│   ├─ 관련 법령
│   └─ 검토 의견
└─ [쟁점 3]
    └─ ...
```

**질의가 여러 개인 경우:**
```
**2. 본 법무법인의 검토의견**
  **가. 질의 (1)에 대한 검토**
    (1) 벤처투자법상 제한 여부
      가) 관련 법규
      나) 검토 의견
    (2) 은행법상 제한 여부
      가) 관련 법규
      나) 검토 의견
  **나. 질의 (2)에 대한 검토**
  **다. 질의 (3)에 대한 검토**
```

### 3.5 결론 (Conclusion)

- 별도 섹션으로 분리하거나, 검토의견 마지막 단락에서 종합.
- "따라서" 또는 "위의 사실관계와... 종합해 보면"으로 시작.
- 핵심 결론을 명확하게 재진술.
- 필요한 경우 추가 확인 필요 사항 부기: "다만, ... 추가 확인이 필요할 것으로 판단됩니다."

## 4. Statute Block Format (법령 인용 형식)

### 4.1 법령 블록 (Statute Block)

관련 법령은 **테두리 있는 박스(테이블)** 안에 전문을 인용합니다:

```
┌──────────────────────────────────────────────┐
│ [법률명]                                       │
│                                                │
│ 제X조(조문 제목)                                │
│ ① 조문 내용...                                 │
│   1. 호 내용...                                │
│     가. 목 내용...                              │
│                                                │
│ [시행령명]                                      │
│ 제X조(조문 제목)                                │
│ ...                                            │
└──────────────────────────────────────────────┘
```

**규칙:**
- 법률명은 **볼드체 + 대괄호**: **[벤처투자법]**, **[은행법]**, **[도로법]**
- 조문 제목도 볼드체: **제39조(벤처투자회사의 행위제한)**
- 항은 원문자(①②③), 호는 아라비아 숫자(1. 2. 3.), 목은 한글(가. 나. 다.)
- 관련 조문만 발췌 가능하나, 발췌 시 "(생략)" 표시
- 하나의 블록 안에 법률 + 시행령 + 시행규칙을 함께 인용 가능 (계층 순서대로)

### 4.2 본문 내 법령 인용 (Inline Citation)

본문에서 법령을 언급할 때:

```
벤처투자법 제39조 제1항 제3호, 동법 시행령 제25조 제2호 및 3호는...
```

**규칙:**
- 법률명에 「꺾쇠 괄호」 사용: 「벤처투자 촉진에 관한 법률」
- 약칭 사용 시 정의 후 사용: "벤처투자 촉진에 관한 법률(이하 '벤처투자법')"
- 동일 법률 재인용: "동법", "같은 법", "동조", "같은 조"
- 조·항·호·목 구분: 제X조 제X항 제X호 X목

## 5. Case Law Citation Format (판례 인용 형식)

### 5.1 판결 인용 (Case Citation)

```
서울고등법원 2016. 5. 31. 선고 2015누71466 판결
대법원 1992. 12. 8. 선고 92다22725 판결
서울행정법원 2020. 9. 29. 선고 2019구합68411 서훈취소결정처분취소의소 판결
```

**형식:** `[법원명] [선고일] 선고 [사건번호] 판결`

**규칙:**
- 선고일: YYYY. M. D. 형식
- 사건번호: 법원 고유 표기 그대로 (예: 2024고정285, 2025나209198)
- 사건명(소송물): 필요 시 사건번호 뒤에 기재
- 복수 판결 인용 시: "대법원 ... 판결, 대법원 ... 판결 등 참조"

### 5.2 판결 내용 인용 (Quoting from Decisions)

판결문 직접 인용 시:

```
서울고등법원은 ... 사안에서,
"(가) 도로는 도로로서의 형태를 갖추고, 도로법에 따른 ... 비로소
도로법의 적용을 받는 도로로 되는 것이므로, ..."고 판시한 바 있습니다.
```

**규칙:**
- 인용문은 큰따옴표(" ")로 감싸기
- 판시 내용이 긴 경우 들여쓰기 또는 별도 단락으로 처리
- 내부 소항목은 (가), (나), (다)로 구분
- 인용 끝에 "~고 판시한 바 있습니다" 또는 "~고 판시하였습니다"

### 5.3 참고 사례 열거 (Case Examples)

여러 참고 사례를 열거할 때:

```
① 간첩조작사건 및 ... 관련하여, ... 원고 패소 판결한 사례
  (서울행정법원 2020. 9. 29. 선고 2019구합68411 판결)

② 논문조작 등을 이유로 ... 원고 일부 승소 판결한 사례
  (서울행정법원 2022. 4. 21. 선고 2021구합50062 판결)

③ 제3자가 ... 각하 판결한 사례
  (서울행정법원 2019. 2. 20. 선고 2018구합4687 판결)
```

- 원문자(①②③)로 번호 매김.
- 사례 요약 + 결론(원고 승/패) → 괄호 안에 판결 정보.

## 6. Defined Terms Convention (정의 용어 관례)

### 6.1 정의 방식

최초 언급 시 전체 명칭을 기술하고 약칭을 정의:

```
주식회사 마이리얼트립(이하 "대상회사"라 합니다)
벤처 블라인드 펀드(이하 "본건 조합")
벤처투자 촉진에 관한 법률(이하 "벤처투자법")
```

**패턴:** `[전체 명칭](이하 "[약칭]"이라 합니다)` 또는 `(이하 "[약칭]")`

### 6.2 정의 용어 작성 규칙

| 대상 | 약칭 패턴 | 예시 |
|------|----------|------|
| 의뢰인 법인 | "귀사", "귀 협회", "귀 기관" | 주식회사 코리아런드리 → 귀사 |
| 상대방/대상 법인 | "대상회사", "경쟁회사" | 주식회사 마이리얼트립 → 대상회사 |
| 자연인 | "표창대상자", "본건 직원" | 이동건 → 표창대상자 |
| 거래/행위 | "본건 출자", "본건 표창" | 장관 표창 → 본건 표창 |
| 계약/관계 | "본건 임대차 계약" | — |
| 사건/분쟁 | "관련 사건", "본건 민원" | — |
| 대상 물건/토지 | "쟁점대지" | — |
| 법률 | 약칭 | 벤처투자 촉진에 관한 법률 → 벤처투자법 |

### 6.3 후속 사용

- 정의 후에는 **약칭만** 사용 (전체 명칭 반복 금지).
- 약칭 사용 시 따옴표 불필요 (첫 정의 시에만 따옴표 사용).

## 7. Tone & Register (문체 및 어조)

### 7.1 기본 원칙

- **경어체(합니다체)** 전면 사용: `~합니다`, `~입니다`, `~사료됩니다`, `~것입니다`.
- **반말/평어 금지:** `~다`, `~이다`, `~한다` 등 반말 어미는 법령 인용문 내부를 제외하고 사용 금지.
- **의뢰인 존대:** "귀사", "귀 협회", "귀 기관" 사용. 의뢰인의 행위를 기술할 때도 존대.
- **자사 겸양:** "저희 법무법인은", "본 법무법인의"

### 7.2 법률 전문 문체 (Legal Register)

한국 법률 의견서 특유의 문어체 표현:

| 표현 | 용법 | 예문 |
|------|------|------|
| ~인바 | "~인데"의 문어체, 선행 사실을 진술할 때 | "위 손해배상금은 ... 감축된 금액이었는바" |
| ~보건대 | "보면" "살펴보면"의 문어체 | "관련 법령을 보건대" |
| ~사료됩니다 | "생각됩니다"의 격식체. 결론 제시 시 | "변상금 납부 의무를 부담하지 않을 것으로 사료됩니다" |
| ~판단됩니다 | 분석적 결론 제시 시 | "적용되지 아니할 것으로 판단됩니다" |
| ~것으로 보입니다 | 추정·불확실한 판단 시 | "무혐의 처분이 내려졌을 것으로 보입니다" |
| ~아니하다 | "않다"의 문어체 | "기소가 이루어지지 아니한 것으로" |
| 나아가 | "furthermore", 추가 논점 전개 시 | "나아가 예상하여 보건대" |
| 가사 | "설령", 양보적 가정 시 | "가사, 본건 출자를 신용공여에 해당하는 것으로 보더라도" |

### 7.3 금지 표현

| 금지 | 대체 |
|------|------|
| ~인 것 같습니다 | ~것으로 판단됩니다 / ~것으로 보입니다 |
| ~일 수도 있어요 | ~가능성이 있습니다 |
| ~해야 될 것 같은데 | ~할 필요가 있을 것으로 판단됩니다 |
| 좀 / 약간 / 아마 | (삭제하거나 정확한 정도 표현으로 대체) |
| 제 생각에는 | (삭제 — 의견서 자체가 법률 의견) |
| ~거든요 / ~잖아요 | (구어체 — 사용 금지) |

## 8. Certainty Language Scale (확신도 표현 체계)

법률 의견서에서는 확신도에 따라 표현을 달리합니다:

### Level 5 — 확정적 결론 (Definitive)
```
~에 해당합니다.
~을 부담합니다.
~은 명백합니다.
```

### Level 4 — 높은 확신 (High Confidence)
```
~것으로 사료됩니다.
~것으로 판단됩니다.
~에 해당하지 않을 것으로 사료됩니다.
~위배되지 않을 것으로 사료됩니다.
```

### Level 3 — 중간 확신 (Moderate Confidence)
```
~것으로 보입니다.
~것으로 해석됩니다.
~가능성이 있습니다.
~여지가 있습니다.
```

### Level 2 — 낮은 확신 / 유보 (Low Confidence / Reserved)
```
~단정하기 어려울 것으로 보입니다.
~보기 어려운 면이 있습니다.
~달리 판단될 수 있습니다.
~인정받기는 어려울 것으로 사료됩니다.
```

### Level 1 — 불확실 / 추가 확인 필요 (Uncertain)
```
~명확히 파악되지 않은 상태입니다.
~추가 확인이 필요할 것으로 판단됩니다.
~파악에는 한계가 있습니다.
```

### 사용 원칙
- 하나의 문서 내에서 동일 쟁점에 대해 확신도가 일관되어야 합니다.
- 결론부에서는 가급적 Level 4 이상을 사용합니다.
- Level 2 이하를 사용할 때는 그 이유(정보 부족, 해석 모호성 등)를 명시합니다.
- "다만" 뒤에 제한 사항을 부기하는 것이 관례입니다:
  ```
  ~것으로 사료됩니다. 다만, ~경우에는 달리 판단될 수 있습니다.
  ```

## 9. Numbering & Enumeration Conventions (번호 매김 관례)

### 9.1 대섹션 (Main Sections)

```
1. 질의의 배경
2. 질의 사항
3. 검토의견
4. 결론
```

- 아라비아 숫자 + 마침표, 볼드체.

### 9.2 중섹션 (Sub-sections)

```
가. 질의 (1)에 대한 검토
나. 질의 (2)에 대한 검토
다. 질의 (3)에 대한 검토
```

- 한글 가나다 순서 (가, 나, 다, 라, 마...).

### 9.3 소섹션 (Sub-sub-sections)

```
(1) 벤처투자법상 제한 여부
(2) 은행법상 제한 여부
(3) 금융지주회사법상 제한 여부
```

- 괄호 아라비아 숫자.

### 9.4 세부 항목 (Detailed Items)

```
가) 관련 법규
나) 검토 의견
```

- 한글 + 닫는 괄호.

### 9.5 본문 내 열거 (Inline Enumeration)

논점을 순차 열거할 때:

```
1) 관련 사건 중 형사 사건은 표창대상자가 아닌 본건 직원에 대한 것이라는 점,
2) 표창대상자가 범죄행위에 개입되었을 경우 ... 기소가 이루어지지 아니한 것으로 보이는 점,
3) 민사 사건에서 사용자 책임이 인정되기는 하였으나 ... 점,
...
등을 고려할 때, ~
```

- 아라비아 숫자 + 닫는 괄호.
- 각 항목 끝에 쉼표(,).
- 마지막 항목 뒤에 "등을 고려할 때" 또는 유사 종결구.

### 9.6 원문자 열거 (Circled Numbers)

판결 내용이나 사례를 열거할 때:

```
① 원고인 경쟁회사의 주장만으로는 ... 배척하였습니다.
② 약관 위반을 근거로 한 ... 판단하였습니다.
③ 본건 직원의 행위는 불법행위에 해당하며 ... 판단하였습니다.
```

- 원문자(①②③...) 사용.
- 주로 판결의 판시사항, 법원의 판단 항목, 사례 열거에 사용.

### 9.7 번호 체계 요약

| 수준 | 형식 | 용도 |
|------|------|------|
| 대섹션 | **1.** **2.** **3.** | 문서 최상위 구조 |
| 중섹션 | **가.** **나.** **다.** | 쟁점별 분류 |
| 소섹션 | **(1)** **(2)** **(3)** | 법률별/논점별 분류 |
| 세부 | **가)** **나)** **다)** | 법령/검토 구분 |
| 본문 열거 | **1)** **2)** **3)** | 논거 나열 |
| 원문자 | **①** **②** **③** | 판시사항, 사례 열거 |

## 10. Closing Disclaimer & Signature Block (종결부)

### 10.1 종결 Disclaimer

문서 본문 종료 후, 서명 전에 반드시 포함:

```
본 Memorandum에 기재된 의견은 귀사를 위하여만 발급되는 것이므로
저희 법무법인은 귀사 이외의 자에 대하여는 이 법률 의견과 관련되어
어떠한 책임이나 부담을 지지 아니하여, 저희 법무법인의 사전 서면동의
없이는 이 법률 의견을 다른 목적이나 용도로 원용하거나 전용할 수
없습니다. 이상의 내용을 귀사의 업무에 참고하시기 바랍니다. 끝.
```

**핵심 요소 (3가지 모두 포함해야 함):**
1. 의뢰인만을 위한 의견이라는 점 (제3자 면책)
2. 무단 전용 금지
3. 업무 참고 요청 + "끝."

**수신자에 맞게 조정:**
- 법인 의뢰인: "귀사"
- 기관/협회: "귀 기관", "귀 협회"
- 개인: "귀하"

### 10.2 본문 종결

결론 또는 마지막 검토의견 끝에 "끝." 기재:

```
... 추가 확인이 필요할 것으로 판단됩니다. 끝.
```

### 10.3 서명 블록

```
법무법인 [법인명]

변호사 [성명]
변호사 [성명]
```

**규칙:**
- 법무법인명 먼저, 한 줄 띄고 변호사 이름.
- 담당 파트너/선임 변호사가 먼저, 주니어 변호사 순서.
- "변호사" 직함 앞에 기재.

### 10.4 각주 (Footnotes)

학술 문헌이나 보충 설명이 필요할 때:

```
[본문]
... 포괄적으로 승인하는 것이 가능하다고 해석되고 있습니다.[1]

[각주]
1. 김홍기, 상법강의, 박영사, 제8판(2024), 544면; 임재연, 회사법Ⅱ,
   박영사, 제9판(2024), 542면; 이철송, 회사법강의, 박영사,
   제32판(2024), 812면 등
```

**학술 문헌 인용 형식:** `저자명, 서명, 출판사, 판수(연도), 면수`
- 복수 문헌은 세미콜론(;)으로 구분.
- "등"으로 다수 문헌 생략 가능.

## 11. Typography & DOCX Generation Rules (타이포그래피)

AI 에이전트가 python-docx 등으로 DOCX를 생성할 때 적용할 규칙:

### 11.1 용지 및 여백
- **용지:** A4 (210mm x 297mm) — 한국 법무법인 표준.
- **여백:** 상하좌우 2.54cm (1 inch).

### 11.2 서체
- **본문 (Latin):** Times New Roman, 11pt
- **본문 (한글/CJK):** 맑은 고딕 (Malgun Gothic), 11pt
- **볼드체:** 섹션 제목, 정보 블록 레이블, 법률명
- **기울임체:** 사용하지 않음 (한국 법률 문서 관례)

### 11.3 CJK 폰트 설정 (python-docx 필수)
```python
# 반드시 w:eastAsia를 명시적으로 설정
from docx.oxml.ns import qn
rPr = run._element.get_or_add_rPr()
rFonts = rPr.get_or_add_rFonts()
rFonts.set(qn("w:eastAsia"), "맑은 고딕")
rFonts.set(qn("w:ascii"), "Times New Roman")
rFonts.set(qn("w:hAnsi"), "Times New Roman")
```

### 11.4 줄간격 및 단락 간격
- **줄간격:** 1.15 (본문)
- **단락 후 간격:** 6pt
- **법령 블록 내부:** 줄간격 1.0, 단락 후 간격 3pt (밀집 배치)

### 11.5 법령 블록 테이블
- 단일 셀 테이블, 테두리 있음 (0.5pt 실선)
- 셀 내부 여백: 상하좌우 5pt
- 배경색: 없음 (흰색) 또는 연한 회색 (#F5F5F5) 선택

### 11.6 색상
- **텍스트:** 검정 (#000000)
- **강조:** 사용하지 않음 (색상 강조 대신 볼드체 사용)
- 보수적 색상 팔레트 유지 — 법률 문서에 컬러 사용은 최소화.

## 12. Writing Flow Patterns (논증 흐름 패턴)

### 12.1 기본 논증 구조

```
[법령 제시] → [법령 해석] → [본건 적용] → [결론]
```

예시:
```
벤처투자법 제39조 제1항 제3호, 동법 시행령 제25조 제2호 및 3호는
... 정하고 있습니다.                              ← 법령 제시

아이엠뱅크가 귀사의 특수관계인인 계열회사에 해당하는 것은
명백하므로, ...                                   ← 법령 해석

그러나, 본 사안에서 아이엠뱅크는 ... 투자자로서의 지위를
갖고 있으며, ...로 해석하기는 어려울 것으로 판단됩니다.  ← 본건 적용

따라서, ... 저촉되지 않을 것으로 사료됩니다.          ← 결론
```

### 12.2 양면 검토 구조 (찬반 구조)

한 쟁점에 대해 찬/반 양면을 모두 검토할 때:

```
[취소 사유가 존재한다고 볼 수 있는 측면]
... 본건 표창을 취소할 이유는 외형적으로 존재한다고 볼 수 있습니다.

[그러나 — 반대 논거]
그러나, 1) ... 점, 2) ... 점, 3) ... 점, ... 등을 고려할 때,
표창대상자에게 본건 표창의 취소 사유가 존재한다고 볼 수는 없다고
볼 것입니다.

[단서/유보]
다만, 위와 같은 판단은 어디까지나 ... 전제로 한 것인데, ...
추가 확인이 필요할 것으로 판단됩니다.
```

### 12.3 전환어 (Transition Words)

| 전환어 | 용도 |
|--------|------|
| 먼저 | 첫 번째 논점 도입 |
| 한편 | 병렬적 다른 논점 전환 |
| 그런데 | 문제 상황 도입 |
| 나아가 | 추가 논점 심화 |
| 그러나 / 그러나, | 반대 논거 전환 |
| 다만 | 제한 사항/유보 조건 |
| 따라서 | 결론 도출 |
| 결국 | 최종 쟁점 정리 |
| 이에 | 앞선 내용의 결과/귀결 |
| 가사 | 양보적 가정 ("설령") |
| 즉 | 부연 설명 / 재진술 |

### 12.4 "~점" 나열 패턴

복수 논거를 나열할 때 한국 법률 의견서 고유의 패턴:

```
1) [논거 A]라는 점,
2) [논거 B]인 점,
3) [논거 C]인 점,
4) [논거 D]이라는 점,
5) [논거 E]할 수는 없는 점,
6) [논거 F]라고 보긴 어려운 점
등을 고려할 때, [결론]
```

**규칙:**
- 각 항목은 "~점,"으로 끝남 (마지막만 "~점"으로 쉼표 생략 가능).
- 전체 나열 후 "등을 고려할 때" 또는 "등에 비추어 볼 때"로 결론 연결.
- 이 패턴은 한국 판결문 스타일을 차용한 것으로, 법률 의견서에서 매우 빈번하게 사용됨.

## 14. Quick Reference Checklist (빠른 참조 체크리스트)

- [ ] 종결 disclaimer + 서명 블록

### 문체 체크
- [ ] 전체 합니다체 (반말/평어 없음)

- [ ] 확신도 표현 적절
- [ ] "다만" 유보 조건 포함 (필요 시)

### 형식 체크
- [ ] 법령 인용: 「법률명」 + 제X조 제X항
- [ ] 판례 인용: 법원명 + 선고일 + 사건번호

- [ ] A4 용지, 적절한 여백
- [ ] 한글 폰트 명시 설정 (맑은 고딕)

## Appendix: Sample Sentence Patterns (예문 패턴집)

실제 의견서에서 추출한 핵심 문장 패턴:

### 사실관계 진술
```
귀사는 [토지/건물]에 대하여 다음과 같이 [계약]을 체결하여 [사업]을 운영 중에 있습니다.
```

### 질의 요약
```
귀사는 위와 같은 상황에서 [쟁점]에 대해 질의하셨습니다.
```

### 법령 해석
```
[법률명] 제X조 제X항은 "[조문 내용]"이라고 정하고 있습니다.
```

### 법령 적용
```
본 사안에서 [주체]는 [행위]로서의 지위를 갖고 있으며, 이에 따라 [결론]로 해석하기는 어려울 것으로 판단됩니다.
```

### 판례 적용
```
위 인정사실에 의하면, [본건 대상]은 [판시 기준]에 해당한다고 보기 어렵고, ... [결론].
```

### 결론 제시
```
따라서, 귀사/임대인은 [쟁점]에 따른 [의무]를 부담하지 않을 것으로 사료됩니다.
```

### 유보 부기
```
다만, 위와 같은 판단은 어디까지나 [전제 조건]을 전제로 한 것인데, [불확실 요소]에 대하여는 추가 확인이 필요할 것으로 판단됩니다.
```

### 양보적 논증
```
가사, [반대 가정]으로 보더라도, [추가 근거]인바, [결론은 동일]할 것입니다.
```

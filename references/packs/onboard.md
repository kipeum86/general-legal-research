# Onboard Reference Pack

This pack contains the detailed onboarding templates, interview, schema, directory,
and completion-message rules. It is loaded only when `.claude/skills/onboard/SKILL.md`
actually runs.

## Existing Config Check

1. Try to read `user-config.json`.
2. If it exists, summarize the main settings and ask `설정을 재구성할까요?`.
3. If the user declines, stop immediately.
4. If the user agrees, continue to onboarding.
5. If it does not exist, continue to onboarding.

## Start Mode Selection

Ask:

```text
설정 방식을 선택해주세요:

[1] 템플릿 선택  — 6개 starter 중 하나를 골라 즉시 생성
[2] 직접 설정    — 인터뷰 (7문항)를 통해 맞춤 설정 생성
```

## Custom Interview

When the user chooses option 2, ask these seven questions in order. If an answer is
missing, apply the default and say `[기본값 적용: ...]`.

1. **이름 / 직함 / 사무소명**
   - Example: `홍길동 / Senior Review Specialist / KP Legal Orchestrator`
   - Default: `[이름 미지정] / Specialist / KP Legal Orchestrator`

2. **주요 업무 분야**
   - Choices: `corporate` / `IP` / `employment` / `regulatory` / `criminal` / `tax` / `other`
   - Default: `corporate`

3. **주요 산업 섹터**
   - Multiple allowed.
   - Examples: fintech, pharma, gaming, real estate, SaaS, healthcare, energy
   - Default: empty list

4. **Primary jurisdiction(s)**
   - Examples: KR, US, EU, SG, JP, CN, UK
   - Default: KR

5. **기본 출력 언어 및 파일 포맷**
   - Language: `ko` / `en` / `ko+en`
   - Format: `docx` / `pdf` / `md`
   - Default: `ko`, `docx`

6. **리서치 기본 깊이**
   - Choices: `Quick` / `Full`
   - Default: `Full`

7. **특수 참조 소스**
   - Optional databases, internal portals, or preferred sources.
   - Default: empty list

## user-config.json Schema

Write this shape at the project root:

```json
{
  "version": "1.0",
  "created": "<오늘 날짜 ISO>",
  "persona": {
    "name": "<이름>",
    "title": "<직함>",
    "firm": "KP Legal Orchestrator",
    "bar_admissions": ["<jurisdiction>"],
    "specialization": "<practice area 요약>"
  },
  "practice": {
    "primary_area": "<primary_area>",
    "sub_areas": [],
    "industry_sectors": [],
    "typical_matters": []
  },
  "jurisdictions": {
    "primary": ["<KR|US|EU|...>"],
    "secondary": []
  },
  "research_defaults": {
    "mode": "D",
    "output_format": "<docx|pdf|md>",

  }

```

## Directory Initialization

### knowledge/

Create:

```text
knowledge/
  _index.md
  statutes/
  cases/
  templates/
  precedents/
```

Initial `knowledge/_index.md`:

```markdown
# Knowledge Base Index

에이전트가 리서치 완료 후 검증된 결과를 여기에 기록합니다.
직접 편집하지 마세요 — 에이전트가 자동으로 관리합니다.

| 파일 | 주제 | 날짜 | 관할 |
|------|------|------|------|
```

## Completion Message

Print:

```text
✓ 설정 완료: {name} @ {firm}
✓ 주요 관할: {primary jurisdictions}
✓ knowledge/ 및 library/ 디렉토리 생성 완료

보유 자료(의견서, 판례, 논문 등)를 library/inbox/에 넣고
/ingest 또는 "자료 넣었어"라고 말하면 자동으로:
- Markdown 변환 → Grade 판별 (A/B/C) → 폴더 배치 → 인덱스 업데이트
Grade A 소스는 리서치 시 최우선 참조됩니다.

이제 리서치 질문을 입력하면 바로 시작됩니다.
```

## Non-Negotiables

1. `user-config.json`, `knowledge/`, and `library/` are local user data and should stay gitignored.
2. Do not overwrite existing user content.
3. Do not proceed to Step 1 until onboarding has produced a usable config.
4. Keep the completion message short after the first setup; avoid repeating the full template/interview text.
